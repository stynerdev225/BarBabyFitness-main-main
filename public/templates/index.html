PDF Templates Setup

This directory is for storing local copies of PDF templates for development purposes.

Required Templates

  • registration_form_template.pdf
  • training_agreement_template.pdf
  • liability_waiver_template.pdf

How to Set Up Templates

You have two options for setting up the templates:

Option 1: Download from Cloudflare R2 (Recommended)

If you have access to the Cloudflare R2 bucket, you can use the provided script to download the templates:

npm run download-templates

This will download the templates from Cloudflare R2 and place them in this directory.

Note: You need to have the correct environment variables set in your .env file:

VITE_CLOUDFLARE_R2_PUBLIC_URL=https://pub-a73c1de02759402f8f74a8b93a6f48ea.r2.dev

Option 2: Manual Download

If you have access to the templates from another source, you can manually download them and place them in this directory.

Why Use Local Templates?

When developing locally, direct browser requests to Cloudflare R2 will be blocked by CORS policies. Using local templates avoids these issues and allows for faster development.

In production, the application will use the templates stored in Cloudflare R2.

Troubleshooting

If you're still experiencing issues with templates:

  • Make sure the file names match exactly as listed above
  • Verify that the files are valid PDF documents
  • Check that the files are placed directly in this directory (not in subdirectories)
  • Restart the development server after adding the templates
